# 参考网站对比分析及修改计划

## 参考网站特点分析 (mcschematic.top)

通过分析参考网站，发现以下关键功能特点：

### 已有功能（参考网站具备）
| 功能 | 描述 |
|------|------|
| **蓝图预览图** | 每个蓝图都有自动生成的预览图（通过 `/api/preview/uuid/xxx` API） |
| **用户头像显示** | 卡片上显示上传者的 QQ 头像 |
| **收藏/点赞系统** | 星星图标表示收藏状态 |
| **排序功能** | "按时间排序" 选项 |
| **蓝图分类** | "公共蓝图"、"TAG检索" 分类 |
| **文件名显示** | 显示完整文件名（如 `xxx.litematic`） |
| **上传者信息** | 显示上传者昵称 |

### 当前项目缺失功能
| 功能 | 当前状态 | 优先级 |
|------|----------|--------|
| 蓝图预览图生成 | 所有 `preview` 字段为空 | **高** |
| 收藏/点赞系统 | 无 | **高** |
| 用户头像显示 | 卡片无头像 | **中** |
| 排序功能 | 无排序选项 | **中** |
| 模态框信息完善 | 缺少版本、模组、上传者信息 | **中** |
| 文件名显示 | 卡片无文件名 | **低** |
| 个人蓝图管理 | 无"我的蓝图"页面 | **低** |

## 修改计划

### 任务 1：完善蓝图卡片和模态框信息显示
**文件**：
- `js/gallery.js` - 修改 `createBlueprintCard()` 和 `openModal()` 函数
- `css/style.css` - 添加头像、版本标签样式

**修改内容**：
- 卡片添加：上传者头像、上传者名称、游戏版本标签、文件名
- 模态框添加：游戏版本、模组依赖、上传者信息、上传时间

### 任务 2：添加收藏/点赞功能
**文件**：
- `js/gallery.js` - 添加收藏状态切换逻辑
- `css/style.css` - 添加收藏图标样式
- `worker-upload.js` - 添加收藏 API（`/favorite`）

**修改内容**：
- 卡片添加收藏按钮（星星图标）
- 实现收藏状态切换
- 收藏数据存储到 localStorage 或 KV

### 任务 3：添加排序功能
**文件**：
- `index.html` - 添加排序下拉选择器
- `js/gallery.js` - 添加排序逻辑

**修改内容**：
- 排序选项：按时间（新→旧）、按时间（旧→新）、按名称、按大小
- 修改 `renderGallery()` 支持排序参数

### 任务 4：蓝图预览图方案（集成第三方服务）
**实施方案**：集成第三方蓝图预览图生成服务

**可选服务**：
- **schematica.xyz** - 提供 NBT/Schematic/Litematic 预览图生成
- **chunkbase.com** - 提供蓝图预览图生成
- **litematic-preview-api** - 开源预览图生成工具

**修改内容**：
- 在 `data.json` 中存储预览图 URL（由第三方服务生成）
- 上传时调用第三方服务 API 生成预览图
- 卡片和模态框显示预览图

## 实施步骤

1. **完善卡片信息显示**（高优先级）
   - 修改 `createBlueprintCard()` 添加版本、作者、文件名信息
   - 修改 `openModal()` 添加完整详情（版本、模组、上传者、上传时间）
   - CSS 添加版本标签样式

2. **添加排序功能**（中优先级）
   - index.html 添加排序选择器（按时间、按名称、按大小）
   - gallery.js 实现排序逻辑

3. **添加收藏功能**（中优先级 - localStorage 方案）
   - 卡片添加收藏按钮（星星图标）
   - 使用 localStorage 存储收藏状态
   - CSS 添加收藏图标样式

4. **集成预览图服务**（高优先级）
   - 调研第三方预览图生成 API（如 schematica.xyz）
   - 上传页面集成预览图生成功能
   - data.json 添加预览图 URL 字段

## 决策确认

- **收藏功能**：✅ 使用 localStorage 存储收藏状态
- **预览图**：✅ 集成第三方蓝图预览图生成服务
- **排序**：默认按上传时间排序（新→旧）
- **头像**：暂不显示上传者头像（需要用户登录后才能获取）
